The Onboarding Flow introduces new users to Huellitas through a three-screen sequence: a branded splash screen, a visually rich welcome screen, and an introduction screen explaining the app's purpose. This flow only appears on first launch.

## Flow Sequence

1. **Splash Screen** (`PantallaCarga`) - 3 seconds, animated logo
2. **Welcome Screen** (`PantallaBienvenida`) - Visual introduction with photos
3. **Introduction Screen** (`PantallaIntroduccion`) - Feature explanation and navigation choice

## Splash Screen

The splash screen displays the Huellitas logo with a subtle pulse animation:

```kotlin
@Composable
fun PantallaCarga(alTerminarCarga: () -> Unit) {
    val transicionInfinita = rememberInfiniteTransition(label = "logo_pulse")
    val escalaLogo by transicionInfinita.animateFloat(
        initialValue = 0.95f,
        targetValue = 1.05f,
        animationSpec = infiniteRepeatable(
            animation = tween(1200),
            repeatMode = RepeatMode.Reverse
        ),
        label = "escala_logo"
    )

    var textoVisible by remember { mutableStateOf(false) }
    val alfaTexto by animateFloatAsState(
        targetValue = if (textoVisible) 1f else 0f,
        animationSpec = tween(durationMillis = 800)
    )

    LaunchedEffect(Unit) {
        textoVisible = true
        delay(3000L)
        alTerminarCarga()
    }

    Box(
        modifier = Modifier.fillMaxSize().background(Color.White),
        contentAlignment = Alignment.Center
    ) {
        Image(
            painter = painterResource(id = R.drawable.logo_huellitas),
            contentDescription = "Logo Huellitas a Salvo",
            modifier = Modifier.size(220.dp).scale(escalaLogo)
        )
    }
}
```

**Animation Details**:
- **Pulse Effect**: Logo scales from 0.95x to 1.05x in 1.2-second cycles
- **Text Fade-in**: App name fades in over 800ms (currently hidden in the code)
- **Duration**: 3 seconds total before transitioning

**Design**:
- White background
- Centered logo (220dp)
- System bars padding for edge-to-edge display

### Staggered Animations

Elements fade in sequentially for a polished feel:

```kotlin
var logoVisible by remember { mutableStateOf(false) }
var imagenesVisibles by remember { mutableStateOf(false) }
var textoVisible by remember { mutableStateOf(false) }
var botonVisible by remember { mutableStateOf(false) }

LaunchedEffect(Unit) {
    logoVisible = true
    delay(300)
    imagenesVisibles = true
    delay(300)
    textoVisible = true
    delay(200)
    botonVisible = true
}
```

**Sequence**:
1. Logo fades in (600ms duration)
2. Wait 300ms
3. Photos fade in (700ms duration)
4. Wait 300ms
5. Text fades in (600ms duration)
6. Wait 200ms
7. Button fades in (600ms duration)

## First-Launch Detection

The onboarding flow should only show on first launch. This is typically managed in the main navigation or MainActivity:

```kotlin
// Example: SharedPreferences check
val prefs = context.getSharedPreferences("huellitas_prefs", Context.MODE_PRIVATE)
val esPrimeraVez = prefs.getBoolean("es_primera_vez", true)

if (esPrimeraVez) {
    // Show splash -> welcome -> introduction
    prefs.edit().putBoolean("es_primera_vez", false).apply()
} else {
    // Go directly to main app
}
```

(Note: The actual persistence mechanism is not shown in the provided source files.)

## Screen Transitions

Navigation between onboarding screens is handled by callbacks:

```kotlin
// In NavHost or similar
composable("splash") {
    PantallaCarga(alTerminarCarga = { navController.navigate("welcome") })
}

composable("welcome") {
    PantallaBienvenida(alSiguiente = { navController.navigate("introduction") })
}

composable("introduction") {
    PantallaIntroduccion(
        alVerAnimales = { navController.navigate("animal_list") },
        alRegistrarAnimal = { navController.navigate("registro") }
    )
}
```

## Design Considerations

**Color Themes**:
- Splash: White background
- Welcome: Purple gradient (adapts to dark mode)
- Introduction: Material 3 surface colors

**Animations**:
- Splash: Infinite pulse on logo
- Welcome: Staggered fade-in for all elements
- Introduction: No animations (static content)

**Accessibility**:
- All images have `contentDescription` for screen readers
- Button labels are descriptive ("Ver animales", "Registrar animal")
- High contrast between text and backgrounds
