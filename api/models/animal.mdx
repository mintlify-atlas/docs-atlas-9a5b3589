## Overview

The Animal model represents a rescued or found animal in the Huellitas system. It is implemented in:

- **PHP**: `models/Animal.php` (backend model with database operations)
- **Kotlin**: `dto/AnimalDto.kt` (data transfer object for API responses)

## Database Schema

The `animales` table structure:

```sql
CREATE TABLE animales (
    id INT PRIMARY KEY AUTO_INCREMENT,
    nombre VARCHAR(255) NULL,
    id_tipo_animal INT NOT NULL,
    raza VARCHAR(255) NULL,
    descripcion TEXT NULL,
    ubicacion VARCHAR(255) NOT NULL,
    contacto VARCHAR(255) NOT NULL,
    id_estado INT NOT NULL DEFAULT 1,
    fecha_registro DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### Related Tables

**tipos_animal** (Animal Types)
```sql
CREATE TABLE tipos_animal (
    id INT PRIMARY KEY,
    nombre VARCHAR(50) NOT NULL
);
-- Values: 1=Perro, 2=Gato, 3=Otro
```

**estados_animal** (Animal States)
```sql
CREATE TABLE estados_animal (
    id INT PRIMARY KEY,
    nombre VARCHAR(50) NOT NULL
);
-- Example: 1=Activo, 2=Adoptado, 3=Inactivo
```

**imagenes_animal** (Animal Images)
```sql
CREATE TABLE imagenes_animal (
    id INT PRIMARY KEY AUTO_INCREMENT,
    id_animal INT NOT NULL,
    imagen_url VARCHAR(500) NOT NULL,
    es_principal TINYINT(1) DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (id_animal) REFERENCES animales(id) ON DELETE CASCADE
);
```

## Field Definitions

### Core Fields

<ResponseField name="id" type="integer" required>
  Unique identifier, auto-generated by MySQL.
</ResponseField>

<ResponseField name="nombre" type="string | null">
  Animal's name. Optional - can be null for unidentified animals.
</ResponseField>

<ResponseField name="id_tipo_animal" type="integer" required>
  Animal type ID:
  - `1` - Perro (Dog)
  - `2` - Gato (Cat)
  - `3` - Otro (Other)
  
  References `tipos_animal.id`.
</ResponseField>

<ResponseField name="tipo_animal" type="string">
  Animal type name (e.g., "Perro", "Gato"). Joined from `tipos_animal` table.
  
  Only present in API responses, not in create requests.
</ResponseField>

<ResponseField name="raza" type="string | null">
  Breed or race. Optional.
</ResponseField>

<ResponseField name="descripcion" type="string | null">
  Description of the animal's appearance, behavior, or circumstances. Optional.
</ResponseField>

<ResponseField name="ubicacion" type="string" required>
  Location where the animal was found. Required field, cannot be empty.
</ResponseField>

<ResponseField name="contacto" type="string" required>
  Contact information (phone, email, etc.). Required field, cannot be empty.
</ResponseField>

<ResponseField name="id_estado" type="integer" required>
  Animal state ID. Defaults to `1` ("Activo").
  
  References `estados_animal.id`.
</ResponseField>

<ResponseField name="estado" type="string">
  State name (e.g., "Activo", "Adoptado"). Joined from `estados_animal` table.
  
  Only present in API responses.
</ResponseField>

<ResponseField name="fecha_registro" type="string">
  Registration timestamp in MySQL DATETIME format: `YYYY-MM-DD HH:MM:SS`.
  
  Auto-set by database on creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  Last update timestamp. Auto-updated by MySQL on any record modification.
</ResponseField>

<ResponseField name="imagen_url" type="string | null">
  URL of the principal image.
  
  In list endpoints, this is the image marked as `es_principal = 1`.
  
  In single animal queries, the full `imagenes` array is included separately.
</ResponseField>

## SQL Injection Prevention

All database queries use **PDO prepared statements** with bound parameters:

```php
$stmt = $this->db->prepare($query);
$stmt->bindValue(':id', $id, PDO::PARAM_INT);
$stmt->execute();
```

Sort columns and directions are validated against whitelists:

```php
$columnasPermitidas = ['fecha_registro', 'nombre', 'id'];
$orden = in_array($orden, $columnasPermitidas, true) ? $orden : 'fecha_registro';
```

See `Animal.php:40-45`.
