title: List Animals
api: GET /api/animales/listar.php
description: Retrieve a paginated list of animals with optional filtering and sorting
---

## Endpoint

```
GET /api/animales/listar.php
```

## Query Parameters

<ParamField query="limite" type="integer" default="10">
  Maximum number of animals to return per page. Minimum value is 1.
</ParamField>

<ParamField query="pagina" type="integer" default="1">
  Page number for pagination. Minimum value is 1.
</ParamField>

<ParamField query="tipo" type="integer">
  Filter by animal type:
  - `1` - Perro (Dog)
  - `2` - Gato (Cat)
  - `3` - Otro (Other)
  
  If provided, `orden` and `direccion` parameters are ignored.
</ParamField>

<ParamField query="orden" type="string" default="fecha_registro">
  Sort column. Allowed values:
  - `fecha_registro` - Registration date
  - `nombre` - Animal name
  - `id` - Animal ID
  
  Only used when `tipo` is not specified.
</ParamField>

<ParamField query="direccion" type="string" default="DESC">
  Sort direction:
  - `DESC` - Descending
  - `ASC` - Ascending
  
  Only used when `tipo` is not specified.
</ParamField>

## Implementation Details

### Query Logic

The endpoint implements two query modes (see `listar.php:27-35`):

1. **Type filter mode**: If `tipo` parameter is provided, returns animals of that type, ignoring `orden` and `direccion`
2. **General mode**: Without `tipo`, returns all animals sorted by specified column and direction

### SQL Injection Prevention

Sort columns and directions are validated against whitelists:

```php
$columnasPermitidas = ['fecha_registro', 'nombre', 'id'];
$direccionesPermitidas = ['ASC', 'DESC'];
```

See `Animal.php:40-45` for implementation.

### Pagination Calculation

```php
$desplazamiento = ($pagina - 1) * $limite;
```

Example: Page 3 with limit 10 → offset 20

### Database Joins

The query joins three tables:
- `animales` (main table)
- `tipos_animal` (animal types)
- `estados_animal` (animal states)
- `imagenes_animal` (images, LEFT JOIN for principal image)

See `Animal.php:47-66` for the full SQL query.
