## Endpoint

```http
POST /api/animales/crear.php
```

## Headers

```http
Content-Type: application/json
```

## Request Body

<ParamField body="id_tipo_animal" type="integer" required>
  Animal type ID:
  - `1` - Perro (Dog)
  - `2` - Gato (Cat)
  - `3` - Otro (Other)
  
  Must be between 1 and 3.
</ParamField>

<ParamField body="ubicacion" type="string" required>
  Location where the animal was found. Cannot be empty.
</ParamField>

<ParamField body="contacto" type="string" required>
  Contact information (phone, email, etc.). Cannot be empty.
</ParamField>

<ParamField body="nombre" type="string">
  Animal's name (optional).
</ParamField>

<ParamField body="raza" type="string">
  Breed or race (optional).
</ParamField>

<ParamField body="descripcion" type="string">
  Description of the animal, including appearance, behavior, or circumstances (optional).
</ParamField>

<ParamField body="imagen_url" type="string">
  URL of the principal image (optional). Typically obtained from the [upload image endpoint](/api/endpoints/upload-image) first.
</ParamField>

## Response

<ResponseField name="status" type="boolean" required>
  `true` if the animal was created successfully, `false` otherwise
</ResponseField>

<ResponseField name="message" type="string" required>
  Human-readable result message
</ResponseField>

<ResponseField name="data" type="object" required>
  The created animal object with all fields populated, including the new `id`.
  
  See [Animal Model](/api/models/animal) for complete structure.
</ResponseField>

## Validation Rules

### Required Fields

The endpoint validates these fields are present and non-empty (see `crear.php:36-48`):

```php
$camposObligatorios = ['id_tipo_animal', 'ubicacion', 'contacto'];
```

### Type Validation

`id_tipo_animal` must be an integer between 1 and 3:

```php
if ($idTipo < 1 || $idTipo > 3) {
    Response::error('El tipo de animal debe ser 1 (Perro), 2 (Gato) o 3 (Otro).');
}
```

See `crear.php:50-54`.

### String Trimming

All string fields are trimmed of whitespace:

```php
$animal->nombre = isset($input['nombre']) ? trim($input['nombre']) : null;
```

Empty strings after trimming are treated as `null`.

## Implementation Details

### Transaction Safety

The creation uses a database transaction to ensure atomicity (see `Animal.php:158`):

1. Begin transaction
2. Insert animal record
3. If `imagen_url` is provided, insert into `imagenes_animal` table
4. Commit transaction

If any step fails, the entire operation is rolled back.

### Auto-Increment ID

The `id` is auto-generated by MySQL:

```php
$animalId = (int) $this->db->lastInsertId();
```

### Default Values

- `id_estado` defaults to `1` ("Activo")
- `fecha_registro` and `updated_at` are auto-set by MySQL `CURRENT_TIMESTAMP`

### Response Data

After creation, the endpoint fetches the complete animal record with JOINs (see `crear.php:67-69`):

```php
$nuevoId = $animal->crear();
$animalCreado = $animal->obtenerPorId($nuevoId);
Response::success($animalCreado, 'Animal registrado correctamente', 201);
```

This ensures the response includes joined data like `tipo_animal` and `estado` names.
