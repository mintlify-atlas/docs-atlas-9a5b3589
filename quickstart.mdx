# Quickstart Guide

This guide will help you get Huellitas up and running quickly. You'll have the app running on an emulator or physical device in just a few minutes.

## Prerequisites

Before you begin, ensure you have:

- **Android Studio Ladybug** or newer installed
- **JDK 11** or higher configured
- An Android device or emulator with **API 24+**

<Note>
For detailed installation requirements and troubleshooting, see the [Installation Guide](/installation).
</Note>

## Get Started

<Steps>
  <Step title="Clone the Repository">
    Clone the Huellitas repository from GitHub:
    
    ```bash
    git clone https://github.com/Alfredogc21/huellitas.git
    ```
    
    Navigate to the project directory:
    
    ```bash
    cd huellitas
    ```
  </Step>

  <Step title="Open in Android Studio">
    1. Launch **Android Studio**
    2. Select **File > Open**
    3. Navigate to the cloned `huellitas` directory
    4. Click **OK** to open the project
    
    Android Studio will automatically detect the Gradle configuration.
  </Step>

  <Step title="Sync Gradle Dependencies">
    Android Studio should automatically start syncing Gradle dependencies. If not:
    
    1. Click **File > Sync Project with Gradle Files**
    2. Wait for the sync to complete (this may take a few minutes on first run)
    
    <Tip>
    If you encounter sync errors, ensure your Android Studio is up to date and you have an active internet connection.
    </Tip>
  </Step>

  <Step title="Configure Backend URL (Optional)">
    Huellitas connects to a PHP backend. By default:
    
    - **Debug builds** connect to local XAMPP: `http://10.0.2.2/huellitas/`
    - **Release builds** connect to production: `https://webculmapp.com/huellitas/`
    
    To change the production URL, edit `app/build.gradle.kts:28`:
    
    ```kotlin app/build.gradle.kts
    buildConfigField("String", "BASE_URL", "\"https://your-domain.com/huellitas/\"")
    ```
    
    <Warning>
    If running locally, make sure XAMPP is running with the backend in `htdocs/huellitas/`.
    </Warning>
  </Step>

  <Step title="Run the App">
    1. Connect an Android device via USB (with USB debugging enabled) or start an emulator
    2. Click the **Run** button (green play icon) or press `Shift + F10`
    3. Select your target device
    4. Wait for the build to complete and the app to launch
    
    <Note>
    The app requires camera and internet permissions. Grant these when prompted.
    </Note>
  </Step>
</Steps>

## First Use Walkthrough

### Onboarding Experience

On first launch, you'll see:

1. **Splash Screen** - Animated welcome screen with the Huellitas logo
2. **Onboarding Slides** - Introduction to the app's features (shown only once)

<Tip>
The onboarding only appears on the first launch. To see it again, clear app data from device settings.
</Tip>

### Registering Your First Animal

After onboarding, you'll reach the home screen with the animal list.

<Steps>
  <Step title="Navigate to Registration">
    Tap the **+** floating action button on the home screen to open the registration form.
  </Step>

  <Step title="Capture a Photo">
    1. Tap the camera icon to open the integrated camera
    2. Point at the animal and tap the capture button
    3. Confirm or retake the photo
    
    The app uses CameraX for high-quality photo capture.
  </Step>

  <Step title="Fill in Details">
    Complete the registration form:
    
    - **Name** - Give the animal a memorable name
    - **Type** - Select Dog, Cat, or Other
    - **Breed** - Specify breed if known
    - **Description** - Add distinctive features or behavior notes
    - **Location** - Where the animal was found
    - **Contact** - Your phone or email for follow-up
  </Step>

  <Step title="Submit Registration">
    Tap **Register** to save the animal to the database. The photo uploads automatically to the server.
    
    You'll be returned to the home screen where your new registration appears in the list.
  </Step>
</Steps>

## Viewing Registered Animals

The home screen displays all registered animals:

- **Filter by type** - Use chips at the top to filter by All, Dogs, Cats, or Others
- **Sort options** - Choose from Recent, By Date, or Oldest
- **Tap any card** - View full details and zoom into photos

## API Endpoints

The app communicates with these backend endpoints:

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/animales/listar.php` | List animals with pagination, filters, and sorting |
| POST | `/api/animales/crear.php` | Register a new animal |
| POST | `/api/animales/subir_imagen.php` | Upload animal photo |

<Note>
All API calls are handled through Retrofit with automatic JSON serialization via Gson.
</Note>
